# START-beta - Интегрированная среда разработки с AI

## Описание

START-beta - это экосистема из двух приложений:
- **StartIDE** - интегрированная среда разработки с поддержкой нейросети
- **Start Office** - офисное приложение для управления проектами с AI ассистентом

## Основные возможности

### StartIDE
- Редактор кода с подсветкой синтаксиса
- Проводник файлов и папок
- Интеграция с Ollama для получения AI советов
- Автоматическая отправка контекста проекта в нейросеть
- Создание и управление проектами в указанной директории

### Start Office
- Обзор проектов и их структуры
- AI ассистент для вопросов о проектах
- История изменений в проектах
- Синхронизация с проектами StartIDE

## Системные требования

- Python 3.7+
- Docker Desktop (для MongoDB и Ollama)
- Tkinter (обычно включен в Python)
- 4GB+ RAM (рекомендуется для Ollama)

## Установка и запуск

### 1. Клонирование репозитория:
```bash
git clone <repository-url>
cd START-beta
```

### 2. Установка зависимостей:
```bash
pip install -r requirements.txt
```

### 3. Запуск Docker сервисов (MongoDB + Ollama):

**Windows:**
```bash
start-docker.bat
```

**Linux/Mac:**
```bash
docker-compose up -d
```

Это запустит:
- **MongoDB** на порту 27017 (база данных)
- **Ollama** на порту 11434 (AI API)
- **Mongo Express** на порту 8081 (веб-интерфейс БД)

### 4. Загрузка AI модели:
```bash
docker exec -it start_ollama ollama pull llama3.1
```

### 5. Запуск приложений:

**StartIDE:**
```bash
python StartIDE/main.py
```

**Start Office:**
```bash
python StartOffice/main.py
```

## Как это работает

### Интеграция с Ollama

Приложения используют Ollama API для взаимодействия с нейросетью. Контекст проектов отправляется в виде текстовых файлов (.txt) для лучшего понимания нейросетью.

### Обмен данными

- StartIDE автоматически отслеживает изменения в проектах
- Контекст сохраняется в **MongoDB** (структура, файлы, AI диалоги) ⭐ НОВОЕ
- Start Office получает данные из БД в реальном времени
- Нейросеть получает контекст из MongoDB с раздвижным окном (5 сообщений)
- Поддерживается кэширование ответов AI
- Полная история диалогов сохраняется отдельно от активного контекста

### Формат контекста

Контекст для нейросети форматируется в удобный текстовый формат:
```
=== ПРОЕКТ: project_name ===
Путь: /path/to/project

=== СТРУКТУРА ПРОЕКТА ===
📁 folder1/
📄 main.py
📄 config.txt

=== ФРАГМЕНТЫ КОДА ===
Файл: main.py
--------------------------------------------------
[код файла]

=== ИСТОРИЯ ИЗМЕНЕНИЙ ===
- [записи об изменениях]
```

## 🗄️ MongoDB - Хранение данных

### Что хранится в базе данных:

1. **Проекты** - метаданные, настройки, дерево файлов
2. **Содержимое файлов** - код с возможностью обрезки по строкам
3. **AI Контекст** - раздвижное окно из 5 последних сообщений
4. **Полная история** - все диалоги без удаления
5. **Кэш ответов** - автоматически очищается через 24 часа

### Режимы работы с файлами:

**Режим 1: Автоматическая конвертация**
- Все файлы конвертируются в текстовый формат
- Сохраняются в MongoDB
- Поддерживаются: .py, .js, .html, .css, .md, .txt, .json, .xml

**Режим 2: Ограничение по строкам**
- Укажите `max_context_lines` в настройках проекта
- Или задайте при сохранении конкретного файла
- Пример: сохранить только первые 50 строк

**Режим 3: Полный код**
- Сохраняется весь файл целиком
- Рекомендуется для небольших файлов

### Раздвижное окно AI (5 сообщений):

```
[1] Пользователь: Как создать класс?
[2] Ассистент: Используйте ключевое слово class...
[3] Пользователь: А методы?
[4] Ассистент: Методы - это функции внутри...
[5] Пользователь: Как наследовать?

→ Новый запрос

[1] удаляется
[2]→[1], [3]→[2], [4]→[3], [5]→[4]
[5] = новый запрос
```

Полная история сохраняется отдельно и никогда не удаляется!

## Использование

1. **Запуск Docker сервисов:**
   - Запустите `start-docker.bat` (Windows) или `docker-compose up -d`
   - Проверьте http://localhost:8081 (Mongo Express)
   - Загрузите модель: `docker exec -it start_ollama ollama pull llama3.1`

2. **Создание проекта в StartIDE:**
   - Запустите StartIDE: `python StartIDE/main.py`
   - Выберите "Файл" → "Новый проект"
   - Укажите папку для проекта
   - Настройте параметры в MongoDB (строки, расширения)
   - Начните создавать файлы

3. **Использование AI в StartIDE:**
   - Откройте панель "Нейро" (справа)
   - Задайте вопрос о вашем проекте
   - Контекст автоматически загружается из MongoDB
   - Диалог сохраняется с раздвижным окном

4. **Работа в Start Office:**
   - Запустите Start Office: `python StartOffice/main.py`
   - Выберите проект из списка (загружается из MongoDB)
   - Задавайте вопросы AI ассистенту
   - Просматривайте структуру и историю из базы данных

## Требования к Ollama

- Ollama должен быть запущен на `http://localhost:11434`
- Рекомендуемая модель: `llama2`
- Для установки других моделей используйте `ollama pull <model_name>`

## Возможные проблемы

### MongoDB

1. **Ошибка подключения к MongoDB:**
   - Убедитесь что Docker запущен: `docker ps`
   - Перезапустите сервисы: `docker-compose restart`
   - Проверьте логи: `docker-compose logs mongodb`
   - Убедитесь что порт 27017 не занят другим приложением

2. **Данные не сохраняются:**
   - Проверьте права доступа к volumes
   - Убедитесь что используется правильная строка подключения
   - Проверьте `.env` файл

### Ollama

1. **Ошибка подключения к Ollama:**
   - Убедитесь что Docker контейнер запущен: `docker ps | grep ollama`
   - Проверьте что порт 11434 доступен
   - Перезапустите: `docker-compose restart ollama`

2. **Модель не найдена:**
   ```bash
   docker exec -it start_ollama ollama list
   docker exec -it start_ollama ollama pull llama3.1
   ```

3. **Нейросеть не отвечает:**
   - Проверьте логи: `docker-compose logs ollama`
   - Убедитесь что модель загружена
   - Проверьте доступность RAM (минимум 4GB)

### Приложения

1. **Проект не найден в Start Office:**
   - Убедитесь что проект создан в StartIDE
   - Проверьте MongoDB: http://localhost:8081
   - Проверьте таблицу `projects`

2. **Импорт ошибки:**
   - Установите pymongo: `pip install pymongo>=4.6.0`
   - Проверьте `shared/__init__.py`
   - Убедитесь что `.env` файл создан
